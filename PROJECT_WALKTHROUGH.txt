1. PROJECT ARCHITECTURE
------------------------------------------------------------------------------
The system is designed as an end-to-end Machine Learning pipeline:
[Raw Data] -> [Feature Engineering] -> [Model Training] -> [Optimization API] -> [Dashboard]

It goes beyond simple price prediction ("How much is this book?") to strategic
optimization ("What price maximizes my revenue and chance of being a bestseller?").

------------------------------------------------------------------------------
2. DATA GENERATION (generate_sample_data.py)
------------------------------------------------------------------------------
We synthesize a realistic dataset of 1,000+ books to simulate market dynamics.

- **Sales Rank Logic**: We simulate "Success" (Sales Rank) based on multiple factors:
  * Author Reputation (Famous authors get better ranks)
  * Price Elasticity (Very high/low prices hurt rank)
  * Title Keywords (e.g., "Ultimate Guide" boosts rank)
  
- **NLP Signals**: Titles are generated with semantic structure (Prefix + Topic + Suffix)
  to allow our model to learn that "The Ultimate Python Bible" sells better than "Python Book".

------------------------------------------------------------------------------
3. ADVANCED FEATURE ENGINEERING (src/feature_engineering_advanced.py)
------------------------------------------------------------------------------
Before training, we transform raw data into powerful signals:

- **NLP Features**:
  * `title_len_char`: Length of title (Longer titles often imply value).
  * `title_has_Ultimate`, `title_has_Guide`: Binary flags for high-converting keywords.

- **Target Encoding**:
  * `author_freq`: How many bestsellers has this author written? (Proxy for fame).
  * `genre_target_price`: The average price for this specific genre.

- **Success Target**:
  * `is_success`: We calculate the Top 25% of Sales Rank and mark these books as "1" (Success).
  This turns a regression problem into a classification problem for probability estimation.

------------------------------------------------------------------------------
4. MULTI-MODEL TRAINING (src/train_advanced.py)
------------------------------------------------------------------------------
We train TWO separate models using CatBoost (Gradient Boosting on Decision Trees):

- **Model A: Price Regressor (Predicts "Market Price")**
  * Goal: What does the market expect this book to cost?
  * Features: Genre, Author, Title, Ratings (BUT NOT PRICE).
  * Output: A dollar amount (e.g., $24.99).

- **Model B: Success Classifier (Predicts "Bestseller Probability")**
  * Goal: Will this book be a bestseller at a given price?
  * Features: Genre, Author, Title, Ratings, AND PRICE.
  * Output: A probability (0.0 to 1.0).
  * *Critical Detail*: By including "Price" as a feature, this model learns Price Elasticity.

- **Explainability (SHAP)**:
  * We calculate SHAP values to explain *why* a price was predicted.
  * Example: "Being a 'Hardcover' added +$5.00 to the prediction."

------------------------------------------------------------------------------
5. CAUSAL OPTIMIZATION ENGINE (src/predict_advanced.py)
------------------------------------------------------------------------------
This is the "Brain" of the application. It doesn't just guess; it simulates.

Step-by-Step Logic:
1. Take a book's profile (Title, Author, Genre).
2. Simulate 20 different price points ranging from $4.99 to $50.00.
3. For EACH price point, ask Model B: "What is the probability of success at this price?"
4. Calculate **Expected Value** = Price * Probability(Success).
5. The price with the highest Expected Value is the **Optimal Strategy**.

Result: A curve showing where you maximize revenue without killing demand.

------------------------------------------------------------------------------
6. INTERACTIVE DASHBOARD & API
------------------------------------------------------------------------------
- **Dashboard (app.py)**:
  * Built with Streamlit for interactivity.
  * **Strategy Simulator**: Visualizes the Optimization Curve (Price vs. Success).
  * **Deep Dive Analytics**: Explores the raw data and distributions.
  
- **API (src/api.py)**:
  * Built with FastAPI for high-performance predictions.
  * Endpoint: `/predict` accepts JSON and returns Market Price + Optimal Price.

------------------------------------------------------------------------------
7. HOW TO RUN EVERYTHING
------------------------------------------------------------------------------

1. **Start the Dashboard**:
   Double-click `start_dashboard.bat` (Windows)
   OR run: `python -m streamlit run app.py`

2. **Start the API**:
   Run: `python -m uvicorn src.api:app --reload`

3. **Re-Train Models** (Optional):
   Run: `python src/train_advanced.py`

4. **Generate Data Report** (Optional):
   Run: `python src/generate_report.py`
